# OpenView: Empowering MLLMs with Out-of-View VQA
---

OpenView consists of three components:

| Component | Description |
|-----------|-------------|
| **OpenView Pipeline** | A 4-stage automated pipeline for generating high-quality OOV VQA data |
| **OpenView-Dataset** | ~16K training QA pairs generated by the OpenView pipeline from 5 panorama sources |
| **OpenView-Bench** | A curated evaluation benchmark with two out-of-view question types |

### Question Types

- **Contextual**: Given a perspective view, judge which objects, actions, or scenarios are plausible *outside* the visible area.
- **Directional**: Given a view and a rotation instruction (e.g., "turn left 40°"), predict what would come into view.

## OpenView Pipeline

The pipeline uses [vLLM](https://github.com/vllm-project/vllm) with `Qwen/Qwen2.5-VL-72B-Instruct` to automatically generate OOV VQA data from raw panoramas. Configure `inference/configs/batch_config.yaml` before running.

> **Before running any stage**, review and edit the configuration:
> - **Stage 1**: paths and model settings are hardcoded at the top of `batch_stage1.py` (`data_root`, `meta_file`, `output_jsonl`, `model_id`, `tensor_parallel_size`, `gpu_memory_utilization`, etc.).
> - **Stages 2–4**: edit `configs/batch_config.yaml` (`data_dir`, `out_dir`, `cache_dir`, `annotation_path`, `tensor_parallel_size`, `gpu_memory_utilization`, `batch_size`, etc.).

The pipeline runs in four sequential stages. After editing the config, you can run all stages at once:

```bash
cd inference
bash scripts/script_construct.sh
```

### Converting to SFT Format

After running the pipeline, convert proposals to the training conversation format and extract question view images:

```bash
cd dataset
python convert_to_format.py --input_file ../inference/output/xxx_output.json
```

## Inference and Evaluation on OpenView-Bench

All inference and evaluation scripts are in `inference/scripts/`. Edit the `## Modify this` fields in each script before running.

| Script | Description |
|--------|-------------|
| `scripts/script_inf.sh` | Local inference (single GPU) for Qwen-VL, InternVL, LLaVA-NeXT |
| `scripts/script_multi_inf.sh` | Multi-GPU inference via SLURM |
| `scripts/script_eval.sh` | Evaluation: choice accuracy + reasoning accuracy (requires `OPENAI_API_KEY`) |

For API-based inference (OpenAI, Gemini, or a local vLLM server), set the API key, base URL, and model name directly in `inference_api.py`.

Evaluation reports two metrics:
- **Choice Accuracy**: whether the selected option (A–E) matches the ground truth.
- **Reasoning Accuracy**: LLM-judged evaluation of per-option rationale quality.
